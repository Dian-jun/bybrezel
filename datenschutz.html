개인정보처리방침

Brezel Money는 사용자의 금융 데이터와 계정 정보를 투명하고 신중하게 처리합니다.

1. 운영자 정보

서비스명: Brezel Money

웹사이트: bybrezel.com/money

이메일: hello@bybrezel.com

2. 처리하는 데이터

✓ Brezel Money는 비로그인 상태에서는 데이터를 사용자의 브라우저 localStorage에 저장하고, 로그인 상태에서는 일부 데이터를 Supabase 데이터베이스와 동기화하여 저장할 수 있습니다.

서비스 이용 과정에서 다음 데이터가 처리될 수 있습니다.

  • 계정 정보: 이메일 주소, 인증 상태, 로그인 관련 정보
  • 가계부 데이터: 거래 내역, 계좌 정보, 고정 지출/수입, 사용자 설정, 카테고리, 예산, 환율 관련 설정
  • 기기 내 저장 데이터: 비로그인 사용 또는 앱 지속 사용을 위한 localStorage 데이터
  • 자연어 입력 텍스트: 선택적으로 외부 API를 통해 파싱될 수 있는 텍스트
  • 기술적 요청 데이터: 환율 조회, 로그인 처리, 보안 및 세션 유지에 필요한 기술 정보

3. 저장 방식 및 동기화

Brezel Money는 로그인 여부에 따라 저장 구조가 달라집니다.

  • 비로그인 상태: 데이터는 브라우저의 localStorage에 저장됩니다.
  • 로그인 상태: 데이터는 localStorage와 함께 Supabase 데이터베이스에도 저장되거나 동기화될 수 있습니다.
  • 최초 로그인 시: 기존 localStorage 데이터가 있는 경우, 이를 Supabase로 자동 마이그레이션할 수 있습니다.
  • 로그인 후: Supabase에 저장된 데이터가 다시 기기에 로드되어 앱에 반영될 수 있습니다.

즉, 현재 Brezel Money는 비로그인 사용자는 로컬 저장, 로그인 사용자는 로컬 저장 + 클라우드 동기화 구조로 동작합니다.

4. 인증 및 계정

Brezel Money는 다음 인증 방식을 제공할 수 있습니다.

  • 이메일 + 비밀번호 로그인
  • 이메일 + 비밀번호 회원가입
  • 비밀번호 재설정 이메일 발송
  • Google 계정을 통한 로그인 또는 가입

로그인은 선택 사항입니다. 사용자는 로그인하지 않고도 앱을 사용할 수 있습니다. 다만 로그인 시 데이터 동기화 및 복원 기능이 제공될 수 있습니다.

5. 제3자 서비스 및 수신자

Supabase (인증 및 데이터 저장)

Brezel Money는 사용자 인증, 계정 관리 및 로그인 사용자의 데이터 저장과 동기화를 위해 Supabase를 사용합니다. Supabase는 당사의 지시에 따라 데이터를 처리하는 처리자(Processor) 또는 인프라 제공자로 기능할 수 있습니다.

Supabase 개인정보처리방침: supabase.com/privacy

Anthropic API (자연어 입력 기능)

자연어로 거래 내역을 입력할 경우, 사용자가 입력한 텍스트가 파싱을 위해 외부 API로 전송될 수 있습니다. 이 기능은 선택 사항이며, 직접 수동 입력만 사용하는 경우 해당 전송은 발생하지 않습니다.

예: “rewe에서 50유로 카드로” → 입력 텍스트를 분석하여 구조화된 거래 정보로 변환

Anthropic 개인정보처리방침: anthropic.com/privacy

환율 API

현재 환율 정보를 표시하거나 저장하기 위해 외부 환율 API를 사용합니다. 일반적으로 금융 거래 자체의 상세 내역은 이 요청에 포함되지 않습니다.

Datenschutzerklärung

Brezel Money verarbeitet Finanz- und Kontodaten transparent und mit besonderer Sorgfalt.

2. Verarbeitete Daten

✓ Brezel Money speichert im nicht eingeloggten Zustand Daten lokal im Browser (localStorage) und im eingeloggten Zustand bestimmte Daten zusätzlich synchronisiert in einer Supabase-Datenbank.

Im Rahmen der Nutzung können insbesondere folgende Daten verarbeitet werden.

  • Kontodaten: E-Mail-Adresse, Authentifizierungsstatus und loginbezogene Informationen
  • Haushaltsbuchdaten: Buchungen, Konten, Fixkosten/Fixeinnahmen, Einstellungen, Kategorien, Budget- und Wechselkursdaten
  • Lokal gespeicherte Daten: localStorage-Daten zur Nutzung ohne Login oder zur gerätebezogenen Persistenz
  • Freitexteingaben: Texte, die optional über einen externen Dienst analysiert werden können
  • Technische Verbindungs- und Anfragedaten: soweit dies für Login, Sicherheit, Sitzung und Wechselkursabfrage erforderlich ist

3. Speicher- und Synchronisationskonzept

Die Art der Speicherung hängt davon ab, ob der Nutzer eingeloggt ist oder nicht.

  • Ohne Login: Die Daten werden lokal im Browser des Endgeräts gespeichert.
  • Mit Login: Daten können sowohl lokal als auch in Supabase gespeichert und synchronisiert werden.
  • Beim ersten Login: Bereits lokal vorhandene Daten können automatisch in die Datenbank migriert werden.
  • Nach dem Login: In Supabase gespeicherte Daten können erneut in die App geladen und lokal verfügbar gemacht werden.

Brezel Money arbeitet damit ohne Login lokal und mit Login lokal plus Cloud-Synchronisierung.

4. Anmeldung und Benutzerkonto

Brezel Money kann insbesondere folgende Anmeldemethoden bereitstellen.

  • Anmeldung mit E-Mail und Passwort
  • Registrierung mit E-Mail und Passwort
  • Versand von E-Mails zum Zurücksetzen des Passworts
  • Anmeldung oder Registrierung mit Google

Die Anmeldung ist optional. Die App kann grundsätzlich auch ohne Login genutzt werden. Ein Login ermöglicht jedoch Synchronisierung und Wiederherstellung von Daten.

5. Drittanbieter und Empfänger

Supabase (Authentifizierung und Datenspeicherung)

Für Benutzeranmeldung, Kontoverwaltung sowie Speicherung und Synchronisierung von Daten eingeloggter Nutzer wird Supabase eingesetzt. Supabase kann dabei als Auftragsverarbeiter (Processor) bzw. Infrastruktur-Anbieter auf Weisung des Verantwortlichen tätig werden.

Datenschutzhinweise von Supabase: supabase.com/privacy

Anthropic API (Freitexteingabe)

Bei der Eingabe in natürlicher Sprache kann der eingegebene Text zur Analyse an einen externen Dienst übermittelt werden. Diese Funktion ist optional; bei manueller Eingabe findet eine solche Übermittlung nicht statt.

Beispiel: „50 Euro bei REWE mit Karte“ → Analyse des Textes → Umwandlung in strukturierte Buchungsdaten

Datenschutzhinweise von Anthropic: anthropic.com/privacy

Wechselkurs-API

Zur Anzeige oder Aktualisierung von Wechselkursen wird eine externe Wechselkurs-API genutzt. Die eigentlichen Finanzbuchungsdaten werden dabei in der Regel nicht mitübermittelt.

8. Speicherung, Löschung und Kontolöschung

Der Speicherort und die Art der Löschung hängen von der Nutzung ab.

  • Lokal gespeicherte Daten löschen: über App-Zurücksetzen oder durch Löschen der Website-Daten im Browser
  • Export/Import: Daten können über die in der App bereitgestellten Funktionen exportiert und importiert werden
  • Löschung serverseitig gespeicherter Daten: Synchronisierte Daten eingeloggter Nutzer können auf Anfrage über hello@bybrezel.com gelöscht werden
  • Kontolöschung: Die Löschung des Benutzerkontos kann über hello@bybrezel.com beantragt werden; betroffene Daten werden gelöscht, soweit keine gesetzlichen Aufbewahrungspflichten entgegenstehen

Das Löschen lokaler Browserdaten führt nicht automatisch dazu, dass auch serverseitig gespeicherte Daten eingeloggter Nutzer entfernt werden.
